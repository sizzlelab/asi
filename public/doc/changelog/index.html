ASI / COS Changelog

Version 0.9 - 28.7.2009

  • Changed return values of every resource.
    • All json-returns are now either:
      • empty (eg. { })
      • contain entry field with returned data (eg. { 'entry' : [ item1, item2 ] } or { 'entry' : item })
      • errors as an array in messages field (eg.{ 'messages' => [ 'Some error', 'Other error' ] } )
    • Changes are present at least in:
      • GET / PUT /appdata/user_id/@self/app_id
      • GET / POST /appdata/app_id/@collections
      • GET / POST / PUT / DELETE /appdata/app_id/@collections/id
      • DELETE /appdata/app_id/@collections/id/@items/item_id
  • Changed parameters of individual resources (mainly POSTs and PUTs):
    • POST /appdata/app_id/@collections
      • New collection's parameters must be wrapped inside collection -hash (eg. collection => { title => 'foo', description => 'bar' } )
    • PUT /appdata/app_id/@collections/id
      • Updates to collection must be wrapped inside collection -hash (eg. collection => { title => 'foo', description => 'bar' } )
    • POST /appdata/app_id/@collections/id
      • New item's parameters must be wrapped inside item -hash (eg. item => { content_type => 'text/plain', body => 'foo' } )
