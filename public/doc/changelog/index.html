Version 0.9.0 - 28.7.2009

This version is incompatible with version 0.8.

Changes to return values

The new standard for returned JSON value formats:

  • data in entry: 
    { 'entry' : [ item1, item2 ] }
    or 
    { 'entry' : item }
  • errors as an array in messages: 
    { 'messages' : [ 'Some error', 'Other error' ] }
    Note that in the case of a single error an array of length 1 is returned.

It is legal for the returned JSON to be empty: { }. This means everything is ok but there was nothing to return.

Many resources were changed to adhere to this standard. The most frequent change is from { foo } to { 'entry' : foo }.

Changes to parameters

  • POST /session
    • Parameters wrapped in session.
  • PUT /people/<user_id>/@location
    • Parameters wrapped in location, except for location_security_token.
  • POST /groups
    • Parameters wrapped in group.
  • POST /channels
    • Parameters wrapped in channel.
  • POST /appdata/app_id/@collections
    • Parameters wrapped in collection.
  • PUT /appdata/app_id/@collections/id
    • Parameters wrapped in collection.
  • POST /appdata/app_id/@collections/id
    • Parameters wrapped in item.
